SMSServer Gateways

Introduction

Gateways represent devices or services capable of sending and/or receiving SMS messages.

SMSServer supports the following gateways, which are directly mapped to SMSLib core library's gateways:

  • Serial Modem
  • IP Modem
  • Clickatell

SerialModem (gateways.SerialModem)

This gateway is used for gsm modems connected via a serial port.

Configuration File

This interface requires the following configuration settings:

OptionDescription
portthe comm port to use, for example COM1 or /dev/ttyS0.
baudratethe baudrate, for example 19200.
manufacturerthe manufacturer string, for example Nokia.
modelthe modem string, for example 6310i.
protocolthe protocol in use, either PDU or TEXT.
pinthe SIM PIN.
inbound"yes" if the gateway should be treaded asan inbound gateway, otherwise "no".
outbound"yes" if the gateway should be treaded as an outbound gateway, otherwise "no".

Example:

gateway.0=myModem, SerialModem
myModem.port=COM1
myModem.baudrate=19200
myModem.manufacturer=Nokia
myModem.model=6310i
myModem.protocol=PDU
myModem.pin=0000
myModem.inbound=yes
myModem.outbound=yes

IPModem (gateways.IPModem)

This gateway is used for gsm modems connected via an IP port.

Configuration File

This interface requires the following configuration settings:

OptionDescription
ipthe IP address of the modem, for example 192.168.100.110.
portthe IP port to which the modem is listening, for example 9900.
manufacturerthe manufacturer string, for example Nokia.
modelthe modem string, for example 6310i.
protocolthe protocol in use, either PDU or TEXT.
pinthe SIM PIN.
inbound"yes" if the gateway should be treaded asan inbound gateway, otherwise "no".
outbound"yes" if the gateway should be treaded as an outbound gateway, otherwise "no".

Example:

gateway.0=myModem, IPModem
myModem.ip=192.168.100.110
myModem.port=9900
myModem.manufacturer=Nokia
myModem.model=6310i
myModem.protocol=PDU
myModem.pin=0000
myModem.inbound=yes
myModem.outbound=yes

ClickatellHttp (gateways.ClickatellHttp)

This gateway is used for the Clickatell HTTP interface.

Configuration File

This interface requires the following configuration settings:

OptionDescription
api_idClickatell API ID.
usernameClickatell Username.
passwordClickatell Password.
secureIf set to "yes", the gateway will use a secure (https) connection.
outbound"yes" if the gateway should be treaded as an outbound gateway, otherwise "no".

Example:

gateway.0=myClickatell, ClickatellHttp
myClickatell.api_id=1234
myClickatell.username=thanasis
myClickatell.password=password
myClickatell.secure=no
myClickatell.outbound=yes
